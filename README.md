# Симулятор энергосети (SoA + DP)

## Описание
Проект моделирует работу энергосети на 24 часа.  
Есть потребители (почасовое потребление), солнечные генераторы (переменная генерация, дешёвая) и дизельные генераторы (постоянная мощность, дорогие).

Цели:
1. В каждый час обеспечить всех потребителей, если это возможно.
2. Минимизировать стоимость генерации.
3. Если генерации недостаточно — максимизировать число обслуженных потребителей.

---

## Быстрый старт

Запуск всех тестов:
```bash
python3 main.py
```

Запуск конкретного теста:
```bash
python3 main.py -t excess
python3 main.py -t deficit
```

## Структура данных (SoA)

Вместо массива объектов используется Structure of Arrays (SoA), что улучшает локальность памяти и ускоряет вычисления.

### Consumers
- `ids: List[str]`
- `demand: List[List[float]]` (n x 24)

### SolarGeneratos
- `ids: List[str]`
- `profile: List[List[float]]` (m x 24)
- `cost_per_kwh: List[float]`

### DieselGeneratos
- `ids: List[str]`
- `power: List[float]`
- `cost_per_kwh: List[float]`

## Логика симуляции

### 1. Предвычисления
Для каждого часа:
- суммарная солнечная генерация и её стоимость;
- суммарный спрос.

### 2. Дефицит энергии
Если `max_total_gen < total_demand`, выбирается максимальное число потребителей жадно (минимальные потребления в приоритете).  
Используется частичная сортировка, а не полная сортировка массива.

### 3. Выбор дизелей
Используется **DP** для выбора набора дизелей с минимальной стоимостью при суммарной мощности ≥ нужной.

Стоимость дизеля за час:
```
cost = power * cost_per_kwh
```

---

## Тестовые сценарии

### 1) Избыток генерации
Спрос понижен так, что **в любой час** суммарной генерации достаточно.

### 2) Дефицит энергии
Спрос выше, солнечная генерация слабая — часть потребителей отключается.

## Упущения модели и возможные улучшения
Модель потребления очень примитивная.
Чтобы ее приблизить к реальным обстоятельстам необходимо учитывать, что каждый потребитель должен иметь свой приоритет отключения при нехватке энергии. 
Также, можно было бы оптимизировать модель наличием аккумуляторов. Тогда можно было бы удерживать излишки энергии, чтобы минимизировать отключения при дефиците. 
При наличии аккумуляторов можно было бы собирать статистику потребления и, сделав из нее выводы, составить оптимальный алгоритм включения дешевых дизельных двигателей для накопления энергии.  
Расширение планирования также улучшит модель выработки энергии.
